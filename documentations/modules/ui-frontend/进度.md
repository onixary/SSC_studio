# 界面前端模块进度

## 当前阶段

基础前端框架实现阶段。

## 已完成

- 确定初始技术栈方向。
- 确定界面前端模块边界。
- 创建 Electron + React + TypeScript + Vite 基础工程。
- 实现主菜单页面框架。
- 实现 Power 蓝图页面框架。
- 实现 Electron 目录选择入口。
- 实现 SSC 项目路径校验。
- 实现形态与 Power 数据读取 API。
- 实现新建 Power JSON API，并可追加到当前形态 origin 的 powers 列表。
- 实现未保存蓝图修改的保存/丢弃中断确认流程。
- Electron 窗口已改为无系统边框，并隐藏系统菜单栏。
- 已实现应用内右上角最小化、最大化/还原、关闭按钮。
- 页面外层已改为贴边沉浸式布局。
- Power 蓝图页面的形态列表与 Power 列表已改为局部滚动区域。
- Power 蓝图页面已在形态列表顶端加入“独立 Power”分类，用于显示未登录到任何形态 origin 的 Power。
- 在“独立 Power”分类中新建 Power 时，不会自动追加到任何形态 origin 的 `powers` 列表。
- 已自动保存最后一次验证通过的 SSC 项目根目录，并在下次应用启动时自动校验和恢复。

- `npm run build` 已通过。
- 已使用临时 mock SSC 项目验证项目识别、形态/Power 读取、新建 Power、重复命名拒绝和非法命名拒绝。
