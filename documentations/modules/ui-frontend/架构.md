# 界面前端模块架构

## 初始分层

```text
Electron Main
  - 文件系统访问
  - 项目路径管理
  - 后续本地命令调用

Preload / IPC
  - 暴露受控 API
  - 隔离渲染进程和 Node 能力

React Renderer
  - 应用布局
  - 模块路由
  - 通用 UI
  - 编辑器容器
```

## 建议目录

```text
src/
  main/
  preload/
  renderer/
    app/
    modules/
      power-blueprint/
      form-texture-config/
    shared/
```

## 状态管理

建议全局只维护跨模块状态：

- 当前工作区路径
- 当前打开文件
- 全局诊断信息
- 用户设置

模块内部状态由模块自行管理。

## IPC 原则

- Renderer 不直接访问 Node.js 文件系统。
- 文件读写通过 preload 暴露的受控 API。
- API 应按能力命名，例如 `workspace.listFiles`、`workspace.readTextFile`。
- 写入前应由模块侧提供明确目标路径。
