# 界面前端模块上下文

## 模块定位

界面前端模块负责 SSC_studio 的应用外壳、导航、布局、文件工作区、通用交互组件和跨模块 UI 基础设施。

它不直接承担 Power JSON 的语义转换，也不直接承担贴图处理算法。具体业务逻辑应下沉到对应模块。

## 当前状态

- 项目尚未建立应用代码。
- 初始技术栈规划为 Electron + React + TypeScript + Vite。
- 蓝图画布预计由 Power 蓝图模块基于 React Flow 实现。

## 模块边界

负责：

- 应用窗口与主布局
- 顶部/侧边导航
- 工作区目录选择
- 文件树
- 标签页/编辑区布局
- 通用面板、弹窗、设置页
- 全局主题与基础组件
- Electron IPC 的前端封装

不负责：

- Power JSON 的 AST 语义
- 蓝图节点类型定义
- 贴图 mask 生成算法
- 模组 Java 代码分析

## 关键依赖

- Electron
- React
- TypeScript
- Vite
- Monaco Editor
- 后续可能使用 Zustand 管理全局状态

## 当前风险

- 如果前端模块过早承载业务逻辑，后续 Power 蓝图和贴图配置会难以测试。
- 需要提前规划文件读写权限边界，避免 UI 层直接写入任意路径。

## 下一步

1. 建立 Electron + React + TypeScript 工程。
2. 实现基础布局和工作区选择。
3. 为 Power 蓝图和贴图配置预留模块入口。
