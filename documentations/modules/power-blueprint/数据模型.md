# 形态 Power 蓝图数据模型

## 原则

Power JSON 是外部持久化格式，PowerAst 是内部语义格式，BlueprintGraph 是 UI 交互格式。

不要把 React Flow nodes/edges 直接保存为 power 文件。

## 初始 PowerAst 草案

```ts
type SlotKind =
  | "power"
  | "entity_action"
  | "entity_condition"
  | "item_action"
  | "item_condition"
  | "bientity_action"
  | "unknown";

interface PowerAstNode {
  id: string;
  type: string;
  kind: SlotKind;
  fields: Record<string, PowerAstValue>;
  unknownFields?: Record<string, unknown>;
  sourcePath?: string;
}

type PowerAstValue =
  | string
  | number
  | boolean
  | null
  | PowerAstNode
  | PowerAstValue[]
  | Record<string, unknown>;
```

## 初始 Graph 草案

```ts
interface BlueprintGraph {
  nodes: BlueprintNode[];
  edges: BlueprintEdge[];
  rootNodeId: string;
}

interface BlueprintNode {
  id: string;
  astNodeId: string;
  type: string;
  kind: SlotKind;
  position: { x: number; y: number };
  fieldValues: Record<string, unknown>;
}

interface BlueprintEdge {
  id: string;
  sourceNodeId: string;
  sourceHandle: string;
  targetNodeId: string;
  targetHandle: string;
  slotKind: SlotKind;
}
```

## 未知字段策略

解析时：

- schema 已知字段进入 typed fields。
- schema 未知字段进入 `unknownFields`。

保存时：

- typed fields 按 schema 输出。
- unknown fields 合并回原始对象。
