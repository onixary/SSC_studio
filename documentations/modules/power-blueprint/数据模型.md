# 形态 Power 蓝图数据模型

## 原则

Power JSON 是外部持久化格式，PowerAst 是内部语义格式，BlueprintGraph 是 UI 交互格式。

不要把 React Flow nodes/edges 直接保存为 power 文件。

如果一个 JSON 文件只包含单个 power，顶层必须直接导出该 power；只有一个文件包含多个命名 power 时，才使用 `origins:multiple` 作为顶层容器。内部 AST 可以统一表达根节点，但序列化时必须保留这种语义差异。

`origins:multiple` 只是同一 JSON 中承载多个命名 power 的容器，不应作为可见蓝图节点。Graph 投影应跳过该容器，直接显示其中的每个子 power；从 Graph 导出时，如果同一蓝图包含多个 power，再自动补回 `origins:multiple`。

## 当前 PowerAst 初始实现

```ts
type SlotKind =
  | "power"
  | "entity_action"
  | "entity_condition"
  | "item_action"
  | "item_condition"
  | "bientity_action"
  | "block_action"
  | "block_condition"
  | "damage_condition"
  | "fluid_condition"
  | "biome_condition"
  | "datatype"
  | "unknown";

interface AstNode {
  id: string;
  type: string;
  kind: SlotKind;
  label?: string;
  fields: AstField[];
  unknownFields: AstUnknownField[];
}

interface AstField {
  name: string;
  value: AstValue;
  schema: FieldSchema;
  visible: boolean;
}

type AstValue =
  | string
  | number
  | boolean
  | null
  | AstNode
  | AstNode[]
  | Record<string, unknown>;
```

## 当前 BlueprintGraph 初始实现

```ts
interface BlueprintGraph {
  nodes: BlueprintNode[];
  edges: BlueprintEdge[];
}

interface BlueprintNode {
  id: string;
  astNodeId: string;
  type: string;
  kind: SlotKind;
  title: string;
  color: string;
  position: { x: number; y: number };
  fields: BlueprintGraphField[];
}

interface BlueprintGraphField {
  name: string;
  valueKind: FieldValueKind;
  displayValue: string;
  connected: boolean;
  color: "power" | "action" | "condition" | "datatype" | "scalar" | "unknown";
  inputKind?: "int" | "float" | "string" | "boolean";
  sourceHandleId: string;
}

interface BlueprintEdge {
  id: string;
  source: string;
  sourceHandle: string;
  target: string;
  targetHandle: string;
  fieldName: string;
  color: string;
  order?: number;
}
```

数组字段的多条连线共享同一个字段输出端口，并通过 `order` 保存数组成员顺序。

当前 UI 中已经允许临时修改 React Flow 投影：

- 节点位置可拖动调整，但尚未写入 `.ssc-studio`。
- 新建节点、连线、可选字段增删、选择删除目前是画布内状态。
- 标量参数编辑目前只更新 `BlueprintGraphField.displayValue`，尚未回写 AST。
- slot/datatype 参数块的 `connected` 与连接态 `displayValue` 在当前 UI 中由 React Flow edges 推导；它们是画布投影状态，不应作为独立语义源保存。
- 选择状态使用 React Flow 的 `selected` 投影状态，删除节点时会同步移除相连边。
- 框选使用 partial selection，节点与框选区域局部重叠即会被选中。
- `origins:multiple` 不生成可见节点，多个子 power 会作为多个根 power 节点显示。
- 这些交互结果后续需要进入正式 `BlueprintGraph -> PowerAst -> JSON` 导出链路。

## 参数输入

字段 schema 可通过 `numberKind` 区分 `int` 和 `float`。Graph 投影中使用 `inputKind` 描述节点内输入框行为：

- `int`：只接受整数，小数点不合法。
- `float`：接受整数或小数，提交时按浮点数规范化显示。
- `string`：直接接收输入。
- `boolean`：使用节点内开关控件提交 `true` / `false`，不再通过文本输入。

文本输入提交发生在回车或输入框失焦时；非法输入不更新当前字段值。过长文本参数通过输入框和参数块整体向右延伸展示，不自动换行。

## 上下文 Schema

同一个 `type` 可能在不同 registry 中表示不同 kind，例如 Apoli 的 `origins:and`：

- `action/meta/AndAction.createFactory()` 只声明 `actions`。
- `condition/meta/AndCondition.createFactory()` 只声明 `conditions`。

因此 schema registry 需要按 `type + kind` 建立上下文 schema，而不能只用 `type` 作为唯一键。否则 action meta 会错误出现 condition 参数，condition meta 也会错误出现 action 参数。

## 未知字段策略

解析时：

- schema 已知字段进入 typed fields。
- schema 未知字段进入 `unknownFields`。
- 当前阶段为了不丢字段，未识别但可推断为标量、slot、datatype 的字段也会进入 `fields`，并携带推断 schema。

保存时：

- typed fields 按 schema 输出。
- unknown fields 合并回原始对象。
- datatype `vector` 导出为 `{ x, y, z }`，不会额外写入内部 `type: "vector"`。
- datatype `particle_effect` 如果原始语义是简单字符串，则导出回字符串。

## Datatype 表示

- `vector` 在 AST 中使用内部节点 `type: "vector"`，UI 显示为 `Vector`。
- `particle_effect` 在 AST 中固定使用内部节点 `type: "particle_effect"`，UI 显示为 `Particle Effect`。
- Particle Effect 的实际粒子标识符保存在字段 `type` 中，`params` 作为普通 string 字段显示。
- 简单字符串粒子在导出时仍可回写为字符串，带 `params` 的粒子导出为对象。
