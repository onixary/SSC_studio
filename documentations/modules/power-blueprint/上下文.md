# 形态 Power 蓝图模块上下文

## 模块定位

形态 Power 蓝图模块负责将 `shape-shifter-curse-fabric` 中的 `resources/data/shape-shifter-curse/powers` 能力 JSON 转换为类似 UE 蓝图的可视化节点编辑体验。

该模块是 SSC_studio 的核心复杂模块。

## 当前调研结论

目标模组中 `powers` 目录约有 449 个 JSON 文件。

Power JSON 混合使用：

- `origins:*`
- `shape-shifter-curse:*`

高频类型包括：

- `origins:conditioned_attribute`
- `shape-shifter-curse:scale`
- `origins:action_over_time`
- `origins:multiple`
- `origins:modify_food`
- `shape-shifter-curse:add_sustained_instinct`
- `shape-shifter-curse:charge_action`

Java 侧使用 Apoli/Calio 的：

- `PowerFactory`
- `ActionFactory`
- `ConditionFactory`
- `SerializableData`

因此该模块应优先做 schema/AST 驱动，而不是手写固定表单。

## 模块边界

负责：

- power JSON 读取后的 AST 表示
- AST 与 React Flow graph 的双向转换
- 节点类型、端口类型、连线规则
- power/action/condition 的 schema registry
- JSON roundtrip 测试
- Power 蓝图画布和节点属性编辑

不负责：

- 应用外壳和窗口管理
- 形态贴图处理
- 实际模组运行时执行逻辑
- 修改模组 Java 代码

## 关键约束

- JSON AST 是语义数据源。
- React Flow graph 是交互投影。
- 必须尽量保留未知字段。
- 必须支持嵌套 action/condition。
- 必须区分不同 slot 类型。
