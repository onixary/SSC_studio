# 形态 Power 蓝图模块需求

## 目标

让复杂的 Power JSON 可以通过节点蓝图方式编辑，降低维护成本，并保持与现有 Origins/Apoli/SSC 数据结构兼容。

## 基础需求

- 扫描并列出 `powers/*.json`。
- 打开单个 power JSON。
- 将 power JSON 转换为可视化节点图。
- 支持编辑节点字段。
- 支持添加、删除、替换嵌套 action/condition。
- 支持保存回 JSON。
- 支持 raw JSON 视图和可视化视图对照。

## 类型需求

至少需要支持以下 slot 类型：

- power
- entity action
- entity condition
- item condition
- bientity action

后续可扩展：

- block action
- block condition
- damage condition
- item action

## 首批建议支持类型

- `origins:and`
- `origins:multiple`
- `origins:action_over_time`
- `origins:conditioned_attribute`
- `origins:active_self`
- `shape-shifter-curse:scale`
- `shape-shifter-curse:charge_action`
- `shape-shifter-curse:consume_mana`
- `shape-shifter-curse:has_mana`
- `shape-shifter-curse:fire_web_bullet`

## 验收标准

- 能打开代表性 power JSON 并生成节点图。
- 能保存回 JSON，语义结构不丢失。
- 未识别字段不会被静默删除。
- 对非法连接给出明确提示。
- 至少有 JSON roundtrip 单元测试。
