# 形态贴图配置模块架构

## 初始分层

```text
form-texture-config/
  indexer/
    scanFormTextures.ts
    formResourceTypes.ts
  processing/
    maskProcessAdapter.ts
    previewGenerator.ts
  validation/
    validateFormTextures.ts
  ui/
    FormTextureConfigView.tsx
    TexturePreviewPanel.tsx
    MaskConfigPanel.tsx
```

## 资源索引

该模块应先建立只读资源索引，再进入编辑流程。

资源索引应包含：

- form id
- base texture path
- overlay texture path
- colormask path
- RGB mask paths
- model or geo reference path
- output target path

## 工具适配层

现有 mask 处理脚本不应直接被 UI 调用。

应通过 adapter 封装：

- 输入文件
- 输出文件
- 参数
- 运行日志
- 错误信息

## 预览策略

- 原图和生成结果应能并排预览。
- 写入前应优先输出到临时或预览目录。
- 用户确认后再覆盖或写入目标资源路径。

## 待确认问题

- 是否需要内置图片编辑能力，还是只做配置和预览。
- 是否需要支持透明背景、图层叠加和通道查看。
- 是否需要和 Blockbench `.bbmodel` 或 geo 文件建立关联。
