# 仓库上下文

## 仓库定位

SSC_studio 是为 Minecraft Java 模组项目 `shape-shifter-curse-fabric` 制作的本地开发者编辑器。

目标是简化模组开发过程中较难维护的配置模块，尤其是：

- 形态 Power 蓝图
- 形态贴图配置
- 后续自定义形态数据包/资源包构建流程

目标模组仓库路径：

`F:\MC Modding\Projects\shape-shifter-curse-fabric`

## 总目标

构建一个面向模组开发者的可视化编辑器，使复杂配置能够通过结构化 UI 或蓝图方式编辑，并能安全写回目标模组项目。

编辑器应优先保证：

- 与现有模组数据结构兼容
- 文件写回安全
- JSON roundtrip 稳定
- 模块边界清晰
- 规划和进度可追踪

## 约束条件

- 本仓库当前处于文档规划阶段，尚未建立应用代码结构。
- 默认技术栈为 Electron + React + TypeScript + Vite。
- Power 蓝图模块默认使用 React Flow 作为画布。
- Power JSON/AST 是语义数据源，React Flow graph 是交互投影。
- 不应随意修改目标模组仓库，除非任务明确要求。
- 所有写入操作后续都应限制在用户选择的工作区内。

## 已验证

已完成本地调研：

- 目标模组 `powers` 目录约有 449 个 JSON 文件。
- Power JSON 混合使用 Origins/Apoli 原生类型和 SSC 自定义类型。
- Java 侧使用 `PowerFactory`、`ActionFactory`、`ConditionFactory`、`SerializableData` 注册配置结构。
- 形态资源中存在贴图、colormask、mask 相关资源和处理脚本。

本仓库当前已完成基础前端框架实现：

- `npm run build` 已通过。
- 已用临时 mock SSC 项目验证工作区识别、形态/Power 读取、新建 Power 和重复/非法命名拒绝。
- 已支持自动保存最后一次有效 SSC 项目路径，并在应用启动时自动恢复。
- 已接入 `@xyflow/react`，并实现 Power JSON 到只读蓝图画布的初始链路。
- Power 蓝图页面已支持“独立 Power”分类，目标项目当前识别出 39 个未绑定 Power。
- Power 蓝图画布已具备节点拖动、临时连线、新建节点菜单和可选字段增删的交互骨架，尚未持久化到 JSON 或 `.ssc-studio`。
- `npm run dev` 可启动 Vite dev server，直接运行时显示 `http://127.0.0.1:5173`。
- 当前沙箱中后台保活 dev server 不稳定，交付时未保持长期运行。
- 内置浏览器插件当前会拦截本地地址访问，未完成浏览器截图验证。
