# ADR-0001: 技术栈选择

## 状态

已接受，后续可根据实际开发反馈修订。

## 背景

SSC_studio 是本地开发者工具，需要读取和编辑 `shape-shifter-curse-fabric` 的资源、数据包和配置文件。

当前需求包含：

- 本地文件系统访问
- 项目目录扫描
- JSON 编辑与校验
- 类似 UE 蓝图的节点画布
- 后续可能调用 Gradle 或资源处理脚本
- 面向 Windows 本地开发工作流

## 决策

初始技术栈采用：

- Electron
- React
- TypeScript
- Vite
- React Flow
- Monaco Editor
- Ajv / JSON Schema
- Vitest
- Playwright

## 理由

- Electron 提供成熟的桌面应用能力和 Node.js 文件系统生态，适合本地模组开发工具。
- React 与 TypeScript 适合构建复杂交互界面，并能维护较严格的数据类型。
- React Flow 更适合作为可视化 AST/蓝图编辑画布，能够自定义节点、端口和连线校验。
- Monaco Editor 适合承载 raw JSON、diff、schema 提示和诊断信息。
- Ajv / JSON Schema 适合把 Apoli/Origins/SSC 的配置类型转成可校验结构。

## 备选方案

### Tauri

优势是包体小、安全模型更严格。当前没有优先采用，因为本项目更依赖 Node.js 文件系统、前端工具链和本地开发脚本生态。

### Rete.js

适合带运行时执行、数据流和代码生成的节点系统。当前需求重点是编辑 Power JSON，而不是在编辑器内执行蓝图，因此不作为首选画布。

## 后果

- 应用包体会比 Tauri 更大。
- 可以更直接地使用 Node 生态实现文件、schema、测试和本地工具集成。
- 蓝图模块需要自行维护 AST 与 React Flow graph 的双向转换。
